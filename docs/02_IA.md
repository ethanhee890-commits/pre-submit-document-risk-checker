# IA v0.1

## 1. IA 목표

MVP에서는 사용자가 빠르게 문서를 입력하고, 분석 결과를 이해하고, 문장별 개선 방향을 확인하는 흐름을 우선한다.

## 2. 전체 정보구조

```text
제출 전 문서 리스크 점검기
├─ 랜딩
│  ├─ Hero
│  ├─ 사용 문제
│  ├─ 4대 점검 항목
│  ├─ 사용 사례
│  ├─ 안전한 사용 원칙
│  └─ CTA
│
├─ 검사하기
│  ├─ 문서 유형 선택
│  ├─ 텍스트 입력
│  ├─ 검사 옵션
│  ├─ 개인정보/보관 안내
│  └─ 검사 실행
│
├─ 리스크 리포트
│  ├─ 종합 요약
│  ├─ 문체 자연도
│  ├─ AI 작성 리스크
│  ├─ 출처·인용 리스크
│  ├─ 제출 안정성
│  ├─ 문장별 하이라이트
│  ├─ 개선 제안
│  └─ 제출 전 체크리스트
│
├─ 문서함
│  ├─ 검사 이력
│  ├─ 리포트 목록
│  └─ 삭제/보관 설정
│
├─ 조직 관리
│  ├─ 멤버
│  ├─ 문서 사용량
│  ├─ 감사 로그
│  └─ 보관 정책
│
└─ 설정
   ├─ 개인정보
   ├─ 원문 보관 기간
   ├─ 학습 사용 동의
   └─ 알림
```

## 3. MVP 라우트

| Route | 화면 ID | 화면명 | MVP |
|---|---|---|---|
| `/` | LAND-001 | 랜딩 | 포함 |
| `/check` | CHECK-001 | 문서 검사 입력 | 포함 |
| `/checking` | CHECK-002 | 검사 진행 상태 | 포함 또는 모달 |
| `/report` | REPORT-001~004 | 리스크 리포트 | 포함 |
| `/documents` | DOC-001 | 문서함 | 2차 |
| `/settings` | SETTINGS-001 | 설정 | 2차 |
| `/org` | ORG-001 | 조직 대시보드 | 3차 |

## 4. MVP 사용자 플로우

```text

본문 붙여넣기
↓
검사 실행
↓

↓

↓

```

## 5. Navigation

### 5.1 Header

| 항목 | 동작 |
|---|---|
| 로고 | `/` 이동 |
| 기능 | 랜딩 내 기능 섹션 이동 |
| 사용 사례 | 랜딩 내 사용 사례 섹션 이동 |
| 안전 원칙 | 랜딩 내 정책 섹션 이동 |
| 검사하기 | `/check` 이동 |

### 5.2 Report 내부 탭 또는 앵커

| 탭 | 내용 |
|---|---|
| 요약 | 종합 리스크 및 4축 카드 |
| 문장별 분석 | 하이라이트 문장과 근거 |
| 개선 제안 | 원문/제안 비교 |
| 체크리스트 | 제출 전 확인 항목 |

## 6. 정보 우선순위

리포트 화면에서 정보 우선순위는 다음이다.

1. 이 결과가 절대 판정이 아니라는 안내
2. 종합 리스크 구간
3. 4축 리스크 카드
4. 가장 먼저 봐야 할 위험 문장
5. 개선 제안
6. 체크리스트
7. 저장/다운로드
