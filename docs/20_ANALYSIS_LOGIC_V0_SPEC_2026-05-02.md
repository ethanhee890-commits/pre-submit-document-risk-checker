# 실제 분석 로직 v0 설계

## 1. 목적

Sprint 3의 분석 로직 v0는 샘플 데이터 기반 프로토타입을 실제 리스크 신호 엔진으로 전환하기 위한 설계안이다.

이 문서는 구현 승인을 위한 기준 문서이며, 승인 전에는 실제 AI API 연동이나 사용자 원문 서버 저장을 진행하지 않는다.

## 2. 제품 경계

| 항목 | 정책 |
|---|---|
| 결과 성격 | 작성자 판정이 아닌 제출 전 리스크 참고 지표 |
| 분석 축 | 문체 점검, AI 작성 리스크, 출처·인용 리스크, 제출 안정성 |
| 금지 | 단일 AI 확률, 작성자 단정, 회피 목적 표현 |
| 개선 제안 | 원문과 비교 가능한 대안문 또는 사용자가 직접 채울 슬롯형 문장 |
| 허위 생성 방지 | 없는 경험, 허위 수치, 허위 출처를 생성하지 않음 |

## 3. v0 분석 파이프라인

```text
입력 문서
→ 전처리
→ 문장 분리
→ 문서 유형별 기준 적용
→ 4축 리스크 신호 계산
→ 문장별 근거 추출
→ 개선 제안 생성 또는 슬롯형 보완 안내
→ 리포트 생성
```

## 4. 모듈 설계

| 모듈 | 역할 | v0 구현 방식 |
|---|---|---|
| Preprocessor | 공백, 특수문자, 문장 길이 정리 | 규칙 기반 |
| Sentence Segmenter | 한국어 문장 단위 분리 | 규칙 기반, 추후 형태소 분석 검토 |
| Document Profiler | 문서 유형, 길이, 문장 수, 민감정보 신호 추출 | 규칙 기반 |
| Style Risk Engine | 반복 구조, 추상 표현, 과도한 일반론 확인 | 규칙 기반 점수 |
| AI Writing Risk Engine | 균일한 문장 구조, 일반화된 표현, 작성 흐름 단절 신호 확인 | 규칙 기반 점수 |
| Citation Risk Engine | 수치, 조사, 연구, 일반화 주장에 출처 필요성 표시 | 키워드/패턴 기반 |
| Submission Stability Engine | 목적, 역할, 일정, 결과, 파일 제출 안정성 확인 | 문서 유형별 체크 |
| Evidence Mapper | 리스크 근거를 문장과 연결 | 문장별 라벨링 |
| Suggestion Builder | 비교 가능한 대안문 또는 슬롯형 보완안 생성 | 템플릿+문서 유형별 규칙 |

## 5. 점수와 등급

내부 점수는 UI에 직접 노출하지 않는다.

| 내부 점수 | 사용자 표시 | 의미 |
|---:|---|---|
| 0-39 | 양호 | 현재 기준에서 큰 보완 신호가 낮음 |
| 40-69 | 보완 권장 | 제출 전 일부 보완이 유효함 |
| 70-100 | 우선 확인 | 제출 전 먼저 확인할 근거가 있음 |

## 6. 문장별 근거 기준

| 라벨 | 감지 신호 예시 | 사용자 설명 방향 |
|---|---|---|
| 문체 | 반복 어미, 추상적 평가어, 과도한 일반론 | 문장 호흡과 구체성을 보완 |
| AI 리스크 | 균일한 구조, 맥락 없는 정돈감, 작성 흐름 차이 | 작성 방식을 단정하지 않고 직접 확인 유도 |
| 출처 | 통계, 연구, 대부분, 증가/감소, 수치 주장 | 출처·인용 보완 필요성 안내 |
| 안정성 | 목적, 대상, 일정, 역할, 결과 누락 | 제출 전 확인 항목으로 연결 |

## 7. 개선 제안 정책

| 상황 | 제안 방식 |
|---|---|
| 실제 경험이 필요한 자소서 | 없는 경험을 만들지 않고 실제 경험을 넣을 수 있는 자연스러운 문장 구조 제공 |
| 출처가 필요한 리포트 | 허위 출처를 만들지 않고 출처명/연도/대상 위치를 표시하는 슬롯형 문장 제공 |
| 업무문서 | 담당자, 일정, 산출물이 포함된 예시 문장 제공 |
| 민감정보 | 삭제 또는 마스킹 후의 대안 문장 제공 |
| 짧은 문서 | 제출 목적과 근거를 추가한 완결 문장 예시 제공 |

## 8. 출력 데이터 구조 초안

```ts
type RiskLevel = "low" | "medium" | "high";

interface AnalysisReportV0 {
  id: string;
  documentType: "resume" | "report" | "blog" | "business" | "other";
  title: string;
  overallLevel: RiskLevel;
  summary: string;
  disclaimer: string;
  axes: RiskAxisV0[];
  findings: SentenceFindingV0[];
  actionItems: string[];
  checklist: ChecklistItemV0[];
  meta: {
    engineVersion: "analysis-v0";
    createdAt: string;
    characterCount: number;
    sentenceCount: number;
    inputRetentionMode: "none" | "local" | "server-temporary";
  };
}
```

## 9. 승인 전 금지 사항

- 실제 AI API 호출 구현
- 사용자 원문 서버 저장
- 계정 기반 문서함 저장
- 분석 결과를 작성자 판정처럼 표현
- 외부 제출 가능성, 통과 가능성, 회피성 표현 제공
