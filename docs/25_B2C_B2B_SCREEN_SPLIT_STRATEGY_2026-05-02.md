# 개인/B2B 화면 분리 전략

## 1. 목적

개인 MVP와 기관용 데모는 목적, 권한, 데이터 정책, 리포트 해석 방식이 다르다. Sprint 3에서는 두 흐름을 메뉴와 화면 구조에서 명확히 분리한다.

## 2. 분리 원칙

| 원칙 | 설명 |
|---|---|
| 개인 우선 | 기본 공개 화면은 개인 제출 전 점검 흐름 |
| B2B 숨김 | 기관용 화면은 일반 내비게이션에 노출하지 않음 |
| 정책 분리 | 개인 데이터 정책과 조직 데이터 정책을 별도로 표시 |
| 권한 분리 | B2B는 관리자, 검토자, 제출자 역할을 구분 |
| 리포트 목적 분리 | 개인은 자기 점검, B2B는 운영 현황과 정책 관리 |

## 3. 개인 MVP 정보구조

```text
/
/check
/checking
/report
/pdf-report
/documents
/settings
```

개인 MVP 내비게이션:

- 기능
- 사용 사례
- 안전 원칙
- 문서함
- 설정
- 검사하기

## 4. B2B 후보 정보구조

일반 공개 메뉴에서는 숨긴다.

```text
/org
/org/dashboard
/org/reports
/org/policies
/org/members
/org/audit-logs
```

## 5. 화면 목적 비교

| 화면 | 개인 MVP | B2B |
|---|---|---|
| 랜딩 | 개인 사용자 가치 설명 | 조직용 별도 랜딩 후보 |
| 검사 입력 | 본인 문서 점검 | 조직 정책 기준 점검 |
| 리포트 | 본인 제출 전 보완 | 검토자 코멘트와 정책 준수 확인 |
| 문서함 | 개인 저장 리포트 | 팀/조직 리포트 목록 |
| 설정 | 보관/원문 저장 개인 설정 | 조직 보관기간, 권한, 감사 설정 |
| PDF | 개인 보관용 | 조직 공유/검토용 |

## 6. 권한 모델 초안

| 역할 | 권한 |
|---|---|
| 제출자 | 본인 문서 점검, 본인 리포트 열람, PDF 저장 |
| 검토자 | 할당된 리포트 열람, 코멘트, 체크리스트 확인 |
| 관리자 | 멤버 관리, 보관정책 설정, 감사 로그 확인 |

## 7. 데이터 정책 차이

| 항목 | 개인 MVP | B2B |
|---|---|---|
| 원문 보관 | 기본 미보관 | 관리자 정책 필요 |
| 리포트 보관 | 사용자 선택 | 조직 보관기간 |
| 삭제 | 사용자 직접 삭제 | 사용자/관리자 권한에 따라 삭제 |
| 감사 로그 | 없음 | 필수 후보 |
| 관리자 열람 | 없음 | 역할 기반 제한 |

## 8. 리포트 표현 차이

개인 MVP:

```text
제출 전 본인이 직접 확인할 보완 항목
```

B2B:

```text
조직 정책 기준으로 검토가 필요한 항목
```

B2B에서도 작성자를 단정하지 않는 정책은 유지한다.

## 9. 구현 순서 제안

| 순서 | 작업 |
|---:|---|
| 1 | 개인 MVP 내비게이션 유지 |
| 2 | `/org` 일반 메뉴 숨김 유지 |
| 3 | B2B 라우트는 별도 진입 링크로만 접근 |
| 4 | B2B 화면에 프로토타입/데모 고지 추가 |
| 5 | 역할/권한 매트릭스 문서 확정 |
| 6 | 계정 및 조직 저장 정책 승인 후 구현 |

## 10. 승인 전 금지 사항

- B2B 메뉴를 일반 사용자에게 노출
- 조직 관리자 원문 열람 구현
- 계정/조직 저장 기능 구현
- 실제 사용자 파일럿처럼 B2B 데모 소개
- 작성자 판정/감시 도구처럼 표현
