# 데이터 정책 v1

## 1. 목적

Data Policy v1은 실제 분석 로직 v0와 외부 데모 준비 전에 데이터 처리 범위를 확정하기 위한 정책 문서다.

승인 전에는 실제 AI API 연동과 사용자 원문 서버 저장을 구현하지 않는다.

## 2. 기본 원칙

| 원칙 | 설명 |
|---|---|
| 최소 처리 | 분석에 필요한 범위만 처리한다. |
| 기본 미보관 | 원문은 기본 저장하지 않는다. |
| 기본 비학습 | 사용자 원문과 리포트는 학습 데이터로 사용하지 않는다. |
| 명시적 동의 | 원문 저장, 공유, 조직 열람은 별도 선택이 필요하다. |
| 삭제 가능성 | 저장되는 데이터는 사용자가 삭제할 수 있어야 한다. |
| 목적 제한 | 제출 전 리스크 점검 목적 외 사용을 금지한다. |

## 3. 데이터 분류

| 데이터 | 예시 | 기본 처리 | 저장 여부 |
|---|---|---|---|
| 입력 원문 | 사용자가 붙여넣은 문서 본문 | 분석 처리에만 사용 | 기본 미보관 |
| 분석 결과 | 4축 리스크, 문장 근거, 제안문 | 리포트 표시 | 사용자가 저장 선택 시 보관 |
| 설정 | 원문 보관 여부, 삭제 기준 | 브라우저 또는 계정 설정 | 보관 가능 |
| PDF 출력 데이터 | 리포트 출력용 데이터 | PDF 생성에 사용 | 생성 후 폐기 원칙 |
| 민감정보 신호 | 이메일, 전화번호 등 패턴 | 경고 표시 | 원문과 동일 정책 |
| 데모 샘플 | 사전 제작 문서 | 공개 데모용 | 저장 가능 |

## 4. 처리 모드

| 모드 | 대상 | 서버 전송 | 원문 저장 | 비고 |
|---|---|---|---|---|
| Prototype Demo | 현재 배포본 | 없음 또는 PDF 생성 시 일시 전송 | 없음 | 샘플 데이터 기반 |
| Analysis v0 Stateless | 실제 분석 v0 후보 | 분석 요청 중 일시 전송 가능 | 없음 | 응답 후 원문 폐기 |
| Saved Report | 사용자 저장 선택 | 리포트 저장 가능 | 기본 미포함 | 원문 별도 opt-in |
| B2B Workspace | 향후 기관용 | 조직 정책에 따름 | 관리자 정책에 따름 | 감사 로그 필요 |

## 5. 원문 보관 정책

| 상황 | 기본값 | 사용자 선택 |
|---|---|---|
| 개인 사용자 일반 점검 | 미보관 | 리포트만 저장 |
| 개인 사용자 원문 포함 저장 | 꺼짐 | 명시적으로 켜야 함 |
| 외부 데모 | 샘플 문서만 사용 | 실제 문서 입력 비권장 |
| B2B | 관리자 정책 필요 | 보관 기간과 접근 권한 별도 설정 |

## 6. Stateless 처리 TTL

실제 분석 v0를 서버에서 처리하더라도 원문은 영구 저장하지 않는다.

| 상황 | 원문 보유 범위 | 최대 TTL | 폐기 시점 |
|---|---|---:|---|
| 정상 분석 요청 | 메모리 내 처리 | 응답 완료 즉시 | 리포트 응답 생성 직후 |
| 분석 중 오류 | 오류 재현에 필요한 비식별 메타데이터만 유지 | 원문 5분 이내 | 오류 응답 또는 요청 타임아웃 직후 |
| PDF 생성 요청 | PDF 출력에 필요한 제한 데이터 | PDF 응답 완료 즉시 | PDF bytes 반환 직후 |
| 클라이언트 localStorage | 사용자가 저장한 리포트 또는 원문 opt-in | 사용자 설정 | 사용자가 삭제하거나 정리 실행 |

서버 로그, 분석 큐, 오류 모니터링에는 원문 또는 문장 일부를 남기지 않는다.

## 7. 로그 금지 필드

아래 항목은 서버 로그, APM, 오류 추적, 분석 이벤트에 기록하지 않는다.

- 입력 원문 전체
- 문장 단위 원문 조각
- 개선 제안 전문
- PDF 본문 텍스트
- 사용자가 입력한 이름, 연락처, 주소, 소속
- 마스킹 전 민감정보
- 브라우저 localStorage에 저장된 원문

허용 로그는 요청 시각, route, HTTP status, 처리 시간, 문서 유형, 글자 수/문장 수 같은 비식별 메타데이터로 제한한다.

## 8. 서버 전송 정책

실제 분석 로직 v0가 승인되기 전까지 서버 분석 전송은 구현하지 않는다.

승인 후 허용 가능한 범위:

```text
사용자 입력 → 분석 요청 중 일시 처리 → 리포트 생성 → 원문 폐기
```

서버에 남길 수 있는 항목은 아래로 제한한다.

- 요청 시각
- 문서 유형
- 글자 수와 문장 수 같은 비식별 메타데이터
- 원문을 포함하지 않는 오류 로그
- 사용자가 저장한 리포트 데이터

## 9. PDF 생성 데이터 전송 범위

서버 PDF 생성은 데이터 정책 승인 전에는 QA/제한 기능으로만 유지한다.

| 데이터 | 전송 허용 | 제한 |
|---|---|---|
| 리포트 요약 | 허용 | 4축 결과, 조치 항목 중심 |
| 문장별 근거 | 제한 허용 | 선택된 근거 문장만, 원문 전체 금지 |
| 개선 제안 | 제한 허용 | PDF에 표시할 제안문만 |
| 체크리스트 | 허용 | 체크 상태와 항목명 |
| 메타데이터 | 허용 | 문서 유형, 생성일, 리포트 ID |
| 원문 전체 | 금지 | 별도 승인 전 전송 금지 |

## 10. 민감정보 범주와 마스킹

| 범주 | 예시 | 기본 처리 |
|---|---|---|
| 연락처 | 이메일, 전화번호 | 표시 전 마스킹 권고 |
| 고유식별정보 | 주민등록번호, 여권번호, 운전면허번호 | 입력 즉시 경고, 저장 금지 권고 |
| 주소 | 상세 주소, 우편번호 조합 | 시/군/구 수준 외 마스킹 권고 |
| 소속 | 학교, 회사, 부서, 학번, 사번 | 필요 최소 표시 권고 |
| 계좌/결제 | 계좌번호, 카드번호 | 저장 금지 권고 |
| 건강/가족/민감 이력 | 병력, 가족관계, 징계, 평가 내용 | 데모 입력 금지, 실제 서비스에서는 별도 고지 필요 |

마스킹 예시는 `홍길동@example.com → h***@example.com`, `010-1234-5678 → 010-****-5678`처럼 사용자가 의미를 확인할 수 있는 최소 수준으로 처리한다.

## 11. 외부 AI API 사용 승인 조건

외부 AI API는 별도 승인 전 사용하지 않는다. 승인 검토 시 아래 항목을 문서화해야 한다.

| 검토 항목 | 필수 확인 |
|---|---|
| subprocessors | 처리 업체와 하위 처리자 목록 |
| 데이터 범위 | 원문, 리포트, 메타데이터 중 전송 항목 |
| 비학습 보장 | 사용자 입력이 모델 학습에 사용되지 않는 조건 |
| 보관 기간 | 공급자 측 보관 기간과 삭제 방식 |
| 처리 지역 | 데이터 처리 리전과 국외 이전 여부 |
| 보안 | 전송 암호화, 접근 통제, 로그 정책 |
| 장애 대응 | 실패 시 원문 보관 없이 재시도하는 방식 |

## 12. 학습 데이터 정책

| 항목 | 정책 |
|---|---|
| 원문 학습 사용 | 기본 금지 |
| 리포트 학습 사용 | 기본 금지 |
| 품질 개선용 샘플 사용 | 별도 동의 필요 |
| 외부 데모 입력 | 수집 금지 |

## 13. 사용자 권리

| 권리 | 제품 제공 방식 |
|---|---|
| 삭제 | 문서함 리포트와 원문 포함 데이터를 삭제할 수 있어야 함 |
| 내보내기 | 저장 리포트를 PDF 또는 텍스트 요약으로 받을 수 있어야 함 |
| 원문 미보관 확인 | 리포트 저장 시 원문 포함 여부를 명확히 표시 |
| 보관 동의 철회 | 원문 함께 보관 설정을 끄고 기존 원문 포함 리포트를 삭제할 수 있어야 함 |
| 데이터 처리 설명 | 분석, PDF, 문서함에서 처리 범위를 확인할 수 있어야 함 |

## 14. 사용자 고지 문구 초안

```text
입력한 문서는 점검 결과를 생성하는 동안에만 처리됩니다.
기본 설정에서는 원문을 저장하지 않으며, 사용자가 리포트 저장을 선택해도 원문은 포함되지 않습니다.
학습 데이터 사용은 기본적으로 제공하지 않습니다.
```

원문 저장 옵션을 제공할 경우:

```text
원문 함께 보관을 켜면 이 브라우저 또는 계정 문서함에 원문이 함께 저장됩니다.
민감정보가 포함될 수 있으므로 필요한 경우에만 켜 주세요.
```

## 15. 승인 전 금지 사항

- 사용자 원문 서버 저장 구현
- 원문을 로그로 남기는 구현
- 실제 AI API 호출 구현
- 동의 없는 학습 데이터 사용
- 삭제 불가능한 리포트 저장

## 16. 열린 결정 사항

| 항목 | 선택지 | 현재 권장 |
|---|---|---|
| 개인 리포트 보관 기간 | 수동 정리/7일/30일 | MVP는 수동 정리 우선 |
| 원문 포함 저장 | 허용/금지 | 기본 금지, opt-in 허용 검토 |
| 서버 로그 보관 | 7일/30일/90일 | 원문 없는 오류 로그만 30일 이하 |
| B2B 관리자 열람 | 허용/제한 | 역할 기반 제한 |
