# 디자인 방향 v0.1

## 1. 디자인 컨셉

신뢰감 있는 B2B SaaS + 개인 사용자가 부담 없이 쓰는 제출 전 점검 도구.

## 2. 톤

| 항목 | 방향 |
|---|---|
| 분위기 | 차분함, 신뢰감, 명확함 |
| 시각 언어 | 문서, 리포트, 체크리스트 중심 |
| 배제 | 과한 AI스러운 보라/파랑 glow, 미래적 3D 장식 |
| 정보 표현 | 카드, 라벨, 근거 패널, 비교 뷰 |
| 감정 | 불안 조장보다 안심과 다음 행동 안내 |

## 3. 레이아웃 원칙

| 화면 | 레이아웃 |
|---|---|
| 랜딩 | Hero + 가치 카드 + 사용 사례 + 안전 원칙 |
| 검사 입력 | 좌측 입력/우측 안내 또는 상하 구조 |
| 리포트 | 상단 요약 + 4축 카드 + 문장 분석 + 개선 제안 |
| 문장 분석 | 본문 하이라이트와 상세 패널을 나란히 배치 |
| 모바일 | 요약 → 문장 목록 → 상세 패널 순으로 수직 배치 |

## 4. 컴포넌트 스타일

| 컴포넌트 | 방향 |
|---|---|
| Card | 충분한 여백, 명확한 제목과 설명 |
| Badge | 낮음/보통/높음, 문체/AI/출처/안정성 라벨 |
| Textarea | 긴 한글 입력에 맞춘 넓은 높이 |
| Button | 명확한 대비, 과한 그림자 금지 |
| Toast | 작고 명확한 상태 피드백 |
| Notice | 리포트 상단에 차분하게 표시 |
| Checklist | 완료율 표시 가능 |

## 5. 권장 색상 방향

정확한 hex는 덱스가 구현에서 정하되 아래 원칙을 따른다.

| 용도 | 방향 |
|---|---|
| 안정 | 차분한 green/blue 계열 |
| 주의 | soft amber 계열 |
| 고위험 | 과하지 않은 red/rose 계열 |
| 배경 | off-white 또는 very light gray |
| 텍스트 | 고대비 neutral |
| 라인 | low-contrast border |

## 6. 타이포그래피

| 항목 | 기준 |
|---|---|
| 본문 | 한글 가독성 높은 sans-serif |
| 행간 | 1.5 이상 권장 |
| 긴 문장 | line-height 충분히 확보 |
| 버튼 | 한글 8~12자 버튼 폭 고려 |
| 리포트 | 숫자보다 설명문 가독성 우선 |

## 7. 접근성

- 색상만으로 리스크를 구분하지 않는다.
- 모든 라벨에는 텍스트가 있어야 한다.
- 버튼 focus 상태를 제공한다.
- 모바일 터치 영역은 충분히 크게 만든다.
- `prefers-reduced-motion`을 고려한다.
- 하이라이트 문장은 키보드 접근이 가능하면 좋다.

## 8. 피해야 할 UI

- “AI 97%” 같은 과도한 숫자 중심 UI
- 빨간색으로 도배된 공포 유발 UI
- 탐지기 통과율 게이지
- human score 게이지
- bypass 성공률
- 의미 없는 AI 로봇 일러스트
