# 데이터·보안 정책 초안 v0.1

## 1. 원칙

| 원칙 | 설명 |
|---|---|
| 최소 수집 | MVP에서는 원문을 서버 저장하지 않는 것으로 가정한다. |
| 기본 비학습 | 사용자 입력 문서를 학습에 사용하지 않는다. |
| 짧은 보관 | 향후 저장 기능 도입 시 기본 미보관 또는 7일 이내 삭제를 권장한다. |
| 명확한 안내 | 입력창 근처에 보관정책을 표시한다. |
| 조직 고객 분리 | 조직 도입 시 보관기간, 감사로그, 권한 분리를 제공한다. |

## 2. MVP 데이터 처리

| 데이터 | 처리 |
|---|---|
| 입력 원문 | 클라이언트 mock 분석에만 사용 |
| 분석 결과 | mock 데이터 기반 표시 |
| 리포트 저장 | 실제 저장하지 않고 토스트만 표시 |
| 다운로드 | 실제 PDF 생성하지 않고 준비 중 안내 |
| 계정 정보 | 수집하지 않음 |
| 결제 정보 | 수집하지 않음 |

## 3. UI 고지 문구

```text
MVP에서는 입력한 문서를 실제 서버에 저장하지 않는 것을 기준으로 설계합니다.
정식 서비스에서는 원문 보관 여부와 보관 기간을 사용자가 선택할 수 있도록 제공해야 합니다.
```

또는 실제 구현에서 localStorage를 사용한다면:

```text
입력한 문서는 현재 브라우저에 임시로만 보관됩니다.
브라우저 데이터 삭제 시 함께 삭제될 수 있습니다.
```

## 4. 향후 정식 서비스 정책 권장

| 항목 | 기본값 |
|---|---|
| 개인 원문 보관 | 기본 미보관 또는 7일 |
| 조직 원문 보관 | 관리자 선택 |
| 학습 사용 | 기본 비활성화 |
| 삭제 | 사용자 직접 삭제 가능 |
| 민감정보 | 입력 전 경고 또는 자동 감지 |
| 감사로그 | 조직 플랜에서 제공 |
| 관리자 접근 | 권한 분리 |

## 5. 민감정보 경고 대상

향후 기능에서 아래 패턴은 감지 가능하면 경고한다.

- 주민등록번호
- 전화번호
- 이메일
- 주소
- 계좌번호
- 건강정보
- 회사 내부 기밀 키워드
- 미공개 채용/평가 정보

## 6. 금지

- 사용자 원문을 무단 학습 데이터로 사용
- 보관기간 미고지
- 삭제 불가능한 저장 구조
- API key 또는 token을 문서/노트/Git에 저장
- 개인 문서를 외부 공개 리포트로 전송
