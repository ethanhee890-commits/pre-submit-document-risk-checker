## 3. 제니 지적별 대응

| 제니 지적 | 대응 |
|---|---|
| 분석 신뢰도와 짧은 문서 경고가 부족함 | `analysisConfidence`, `reliabilityNotice`, 200자/500자 기준 추가 |
| 4축 종합 등급 산정이 모호함 | critical-first weighted 산정 규칙과 문서 유형별 가중치 추가 |
| AI 작성 리스크가 판정처럼 보일 수 있음 | 실제 작성자 판정이 아닌 작성 패턴 점검 신호로 명시 |
| 데이터 폐기 시점이 모호함 | 정상 응답 즉시 폐기, 오류 시 최대 5분 이내 TTL 추가 |
| 로그 금지 범위가 부족함 | 원문, 문장 조각, 제안문 전문, PDF 본문, 마스킹 전 민감정보 로그 금지 명시 |
| PDF 데이터 전송 범위가 불명확함 | 리포트 요약, 제한된 근거 문장, 제안문, 메타데이터로 제한 |
| 삭제 UX의 완료 증거가 부족함 | 토스트, 목록 개수 감소, 삭제 상세 접근 차단, localStorage 확인 기준 추가 |
| PDF에 원문이 과도하게 포함될 수 있음 | 최대 3개 근거 문장, 문장당 300자 제한 추가 |
| 데모 샘플이 문제 사례 중심임 | 각 문서 유형마다 양호 샘플 추가 |
| B2B가 평가/감시 도구처럼 보일 위험 | 금지 카피, 사람 검토, 소명/수정 기회, 불이익 방지 원칙 추가 |
