# PDF 운영 UX 설계

## 1. 목적

PDF 리포트는 사용자가 제출 전 보완 내용을 저장하거나 공유하기 위한 핵심 산출물이다. Sprint 3에서는 PDF 저장, 파일명, 고지, 개선 제안, 품질 QA 기준을 운영 수준으로 정리한다.

## 2. PDF 리포트 구성

| 순서 | 섹션 | 필수 여부 |
|---:|---|---|
| 1 | 제품명/문서 유형/생성일 | 필수 |
| 2 | 작성자를 단정하지 않는 참고 지표 고지 | 필수 |
| 3 | 프로토타입 또는 분석 엔진 버전 고지 | 필수 |
| 4 | 지금 해야 할 조치 | 필수 |
| 5 | 4축 리스크 요약 | 필수 |
| 6 | 문장별 주요 근거 | 필수 |
| 7 | 개선 제안 | 필수 |
| 8 | 제출 전 체크리스트 | 필수 |
| 9 | 파일명/생성 정보 footer | 필수 |

## 3. 원문 포함 제한

PDF에는 원문 전체를 기본 포함하지 않는다. 사용자가 리포트를 공유할 때 원문과 민감정보가 과도하게 노출되는 일을 줄이기 위해 문장별 근거에 필요한 일부만 표시한다.

| 항목 | 제한 |
|---|---|
| 원문 전체 | 기본 미포함 |
| 문장별 근거 원문 | 최대 3개 문장 |
| 문장 길이 | 문장당 최대 300자, 초과 시 말줄임 |
| 개선 제안 | PDF에 표시한 제안문만 포함 |
| 민감정보 | 마스킹 후 표시 권장 |

원문 전체 포함 PDF가 필요하다는 요구가 확인되면 데이터 정책, 저장 동의, 공유 책임 고지 승인 후 별도 기능으로 분리한다.

## 4. 개선 제안 PDF 반영 기준

| 기준 | 설명 |
|---|---|
| 원문 포함 | 제한된 선택 문장만 표시 |
| 제안문 포함 | 실제 비교 가능한 대안문 표시 |
| 지시문 단독 금지 | 사용자가 바로 복사해도 어색한 지시문만 남기지 않음 |
| 허위 생성 방지 | 없는 경험, 수치, 출처 생성 금지 |
| 텍스트 검색성 | PDF 텍스트 추출 시 제안문 확인 가능 |

## 5. 민감정보 확인 UX

PDF 저장 전 민감정보 신호가 있으면 확인 단계를 표시한다.

```text
PDF에 연락처, 주소, 소속 등 민감정보가 포함될 수 있습니다.
공유 전 마스킹 상태를 확인해 주세요.
```

필수 처리:

- 이메일, 전화번호, 주소, 고유식별정보 패턴을 사전 감지한다.
- PDF 미리보기 또는 저장 전 안내에서 민감정보 포함 가능성을 알려준다.
- 마스킹된 문장과 원문 표시 여부를 구분한다.
- 고유식별정보로 보이는 값은 PDF 포함을 기본 차단하는 방향을 우선 검토한다.

## 6. 파일명 규칙

권장 파일명:

```text
문서신뢰도리포트_{문서유형}_{YYYYMMDD}.pdf
```

예시:

```text
문서신뢰도리포트_자소서_20260502.pdf
```

파일명 정책:

- 한글 파일명 허용
- 특수문자 제거
- 공백 대신 `_` 사용
- 길이 120자 이하

## 7. 저장 방식

| 방식 | 설명 | Sprint 3 판단 |
|---|---|---|
| 브라우저 인쇄 저장 | 사용자가 직접 PDF 저장 | 기본 유지 |
| 서버 PDF 생성 | 동일 리포트를 서버에서 PDF로 생성 | 데이터 정책 승인 전 QA/제한 기능 유지 |
| 공유 링크 | 서버에 리포트 저장 후 공유 | 보류 |
| 이메일 전송 | PDF 첨부 전송 | 보류 |

서버 PDF 생성 기능은 실제 분석/저장 정책 승인 전까지 운영 기본 기능으로 확장하지 않는다.

## 8. 사용자 고지

```text
PDF에는 리포트 요약, 문장별 근거, 개선 제안, 체크리스트가 포함됩니다.
원문 또는 민감정보가 포함될 수 있으므로 공유 전 내용을 확인해 주세요.
```

프로토타입 데모에서는 PDF 상단 또는 footer에 아래 고지를 넣는다.

```text
이 PDF는 제품 흐름 검증용 프로토타입 리포트입니다.
샘플 문서 기준으로 생성되었으며 실제 제출 판단의 근거가 아닙니다.
```

## 9. 외부 데모 PDF 워터마크

제한적 외부 데모에서 생성하는 PDF에는 다음 중 하나를 적용한다.

| 방식 | 기준 |
|---|---|
| 상단 고지 | 첫 페이지 상단에 프로토타입 데모 고지 표시 |
| footer | 모든 페이지에 "프로토타입 데모 리포트" 표시 |
| 파일명 | `DEMO_문서신뢰도리포트_{샘플ID}_{YYYYMMDD}.pdf` 사용 |

## 10. PDF 품질 QA 기준

| 항목 | 기준 |
|---|---|
| 다운로드 | HTTP 200, `application/pdf` |
| 한글 파일명 | UTF-8 filename 처리 |
| 한글 검색성 | 핵심 문구 텍스트 추출 가능 |
| 한글 복사 | 대체문자 없이 추출 |
| 페이지 | A4 기준, 주요 카드 break-inside 방지 |
| 개선 제안 | PDF 화면과 서버 PDF 모두 포함 |
| 원문 제한 | 전체 원문 미포함, 근거 문장 제한 준수 |
| 민감정보 | 마스킹 안내 또는 차단 동작 확인 |
| 금지어 | 회피성 표현 미검출 |

## 11. 운영 전 결정 사항

| 항목 | 선택지 | 현재 권장 |
|---|---|---|
| 서버 PDF 기본 제공 | 예/아니오 | 데이터 정책 승인 후 제공 |
| PDF에 원문 전체 포함 | 예/아니오 | 아니오, 문장별 근거 원문만 제한 표시 |
| 공유용 링크 | 예/아니오 | 파일럿 전 보류 |
| 조직 로고 삽입 | 예/아니오 | B2B 분리 후 검토 |

## 12. 승인 전 금지 사항

- 사용자 원문을 저장한 뒤 공유 링크로 노출
- 민감정보 포함 PDF를 자동 공유
- 실제 분석 검증 전 "최종 안전"처럼 표현
- PDF를 작성자 판정 문서처럼 구성
