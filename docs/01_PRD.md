# PRD v0.1

## 1. 제품명

제출 전 문서 리스크 점검기

## 2. 제품 정의

한국어 문서를 제출하기 전에 문체, AI 작성 리스크, 출처·인용 리스크, 제출 안정성을 점검하는 문서 신뢰도 관리 서비스.

## 3. 배경

AI 문서 판정기 단독 서비스는 사용 빈도와 결제 명분이 약하고, 회피형 서비스로 오해될 가능성이 크다. 따라서 본 제품은 `AI 판정기`가 아니라 `제출 전 문서 리스크 점검`으로 정의한다.

## 4. 문제 정의

사용자는 문서를 제출하기 전 다음 불안을 가진다.

- 내 글이 너무 AI처럼 보이지 않는가?
- 직접 쓴 글인데 AI로 오탐될 위험은 없는가?
- 출처와 인용이 부족하지 않은가?
- 문체가 제출 맥락에 맞는가?
- 수정 전후 리스크를 설명 가능한 형태로 확인할 수 있는가?

기관/조직 고객은 다음 문제가 있다.

- 많은 문서를 일관된 기준으로 검토하기 어렵다.
- AI 작성 여부를 단정할 수 없지만 위험 신호는 관리해야 한다.
- 오탐으로 인한 분쟁을 줄이기 위해 근거와 작성과정 증빙이 필요하다.
- 외주 원고, 지원서, 과제, 리포트의 신뢰도 관리가 필요하다.

## 5. 목표

### 5.1 MVP 목표

텍스트 입력 기반으로 문서 리스크 리포트를 생성하고, 사용자가 문장별 위험 근거와 개선 방향을 확인할 수 있는 클릭형 웹 프로토타입을 만든다.

### 5.2 사업 목표

- B2C 유입: 학생, 취준생, 블로그/콘텐츠 작성자
- 초기 B2B: 학원/컨설턴트, 콘텐츠팀, 마케팅 에이전시
- 장기 B2B: HR, 교육기관, LMS/API 플랫폼

## 6. 핵심 사용자

| 우선순위 | 고객군 | 핵심 니즈 |
|---:|---|---|
| 1 | 취준생/자소서 작성자 | 제출 전 문체와 AI 작성 리스크 확인 |
| 2 | 대학생/대학원생 | 리포트/과제 제출 전 리스크 확인 |
| 3 | 학원/컨설턴트 | 여러 학생 문서 첨삭·리포트 관리 |
| 4 | 콘텐츠팀/에이전시 | 외주 원고 품질·출처·AI 리스크 검수 |
| 5 | HR/채용팀 | 지원서/과제 전형 신뢰도 참고 리포트 |
| 6 | 교육기관/LMS | 무결성 검토 보조와 작성과정 증빙 |

## 7. 핵심 가치

| 가치 | 설명 |
|---|---|
| 제출 전 안심 | 사용자가 제출 전에 위험 신호를 확인한다. |
| 설명 가능한 리스크 | 단일 점수보다 근거와 후속 조치를 제공한다. |
| 한국어 특화 | 자소서, 리포트, 블로그, 업무문서 문체에 맞춘다. |
| 오탐 방어 | 직접 작성한 글도 오탐될 수 있음을 안내하고 증빙 흐름을 제공한다. |
| B2B 확장성 | 조직별 문서 관리, 리포트, API로 확장 가능하다. |

## 8. 제품 원칙

1. 절대 판정이 아니라 리스크 참고 지표를 제공한다.
2. AI 탐지 회피나 통과 보장을 제공하지 않는다.
3. 단일 AI 확률보다 4축 리스크 리포트를 제공한다.
4. 사용자에게 왜 위험한지와 어떻게 보완할지를 함께 제공한다.
5. 개인정보와 원문 보관을 최소화한다.

## 9. MVP 기능

| 기능 | 설명 | 우선순위 |
|---|---|---:|
| 문서 유형 선택 | 자소서, 리포트, 블로그, 업무문서, 기타 | High |
| 텍스트 입력 | 본문 붙여넣기 | High |
| 검사 실행 | mock 분석 결과 생성 | High |
| 종합 리스크 | 4축 리스크 요약 | High |
| 문장별 하이라이트 | 위험 문장별 라벨과 근거 | High |
| 개선 제안 | 문체/출처/명확성 개선 제안 | High |
| 제출 체크리스트 | 제출 전 확인 항목 | High |
| 리포트 저장 mock | 저장 완료 토스트 | Medium |
| 다운로드 mock | 준비 중 상태 | Medium |

## 10. MVP 성공 기준

| 지표 | 기준 |
|---|---|
| 핵심 플로우 완료 | 랜딩 → 입력 → 검사 → 리포트 확인 가능 |
| 결과 이해도 | 절대 판정으로 오해할 문구가 없어야 함 |
| 상호작용 | 문장 클릭, 체크리스트, 복사 토스트 동작 |
| 상태 처리 | empty/loading/error/success 상태 구현 |
| 반응형 | 모바일에서 주요 정보가 깨지지 않음 |
| 문구 정책 | 금지 표현 0건 |

## 11. 비목표

- AI 탐지기 우회
- 특정 탐지기 통과 보장
- 100% 판정
- 실제 표절 DB 검색
- 실제 AI 모델 성능 검증
- 결제/계정 시스템
- 조직 관리 전체 구현

## 12. 리스크

| 리스크 | 대응 |
