# 외부 데모 샘플 문서 세트

## 1. 목적

제한적 외부 데모는 실제 사용자 문서가 아니라 사전 제작된 샘플 문서로만 진행한다.

목표는 분석 정확도 검증이 아니라 화면 흐름, 리포트 이해도, PDF 가치, 문서함/설정 정책 이해도를 확인하는 것이다.

## 2. 데모 고지

데모 시작 전 아래 문구를 읽거나 화면에 표시한다.

```text
이 화면은 제품 흐름 검증을 위한 프로토타입 데모입니다.
현재 결과는 실제 AI 분석이 아니라 샘플 데이터 기반으로 표시됩니다.
실제 제출 판단이나 작성자 판정의 근거로 사용하지 마세요.
```

## 3. 샘플 문서 원칙

| 원칙 | 설명 |
|---|---|
| 실제 개인정보 없음 | 이름, 전화번호, 이메일, 주소 제외 |
| 실제 제출 문서 아님 | 외부 제출 가능한 실제 문서 사용 금지 |
| 허위 수치 최소화 | 수치는 샘플임을 표시하거나 일반 예시로 사용 |
| 민감 문서 금지 | 채용, 평가, 과제, 내부 보고 원문 사용 금지 |
| 문서 유형별 다양성 | 자소서, 리포트, 블로그, 업무문서 각 1개 이상 |

## 4. 샘플 A: 자소서

목적: 추상 표현과 경험 구체성 부족을 보여준다.

```text
저는 성장 과정에서 여러 경험을 통해 책임감과 문제 해결력을 키워 왔습니다.
특히 팀 활동을 하면서 다양한 의견을 조율했고, 맡은 일을 끝까지 해내는 태도를 배웠습니다.
앞으로도 이러한 장점을 바탕으로 조직에 기여하는 사람이 되고 싶습니다.
```

기대 리포트:

- 문체 점검: 보완 권장
- AI 작성 리스크: 보완 권장
- 출처·인용 리스크: 양호
- 제출 안정성: 우선 확인

## 5. 샘플 B: 리포트

목적: 출처·인용 보완 필요성을 보여준다.

```text
최근 청년층의 디지털 학습 참여는 빠르게 증가하고 있다.
대부분의 대학생은 온라인 자료를 더 편리하게 느끼며, 이러한 흐름은 앞으로도 확대될 가능성이 크다.
따라서 대학은 디지털 학습 환경을 중심으로 지원 정책을 다시 설계해야 한다.
```

기대 리포트:

- 출처·인용 리스크: 우선 확인
- 문장별 근거에서 조사명, 연도, 대상 필요성 표시

## 6. 샘플 C: 블로그

목적: 반복 표현과 넓은 일반론을 보여준다.

```text
이 방법은 매우 좋고 많은 사람에게 도움이 됩니다.
간단하면서도 유용하고, 누구나 쉽게 따라 할 수 있다는 점에서 장점이 큽니다.
여러 상황에서 활용할 수 있어 많은 독자에게 추천할 만합니다.
```

기대 리포트:

- 문체 점검: 우선 확인
- 개선 제안에서 실제 사용 상황 중심 대안문 표시

## 7. 샘플 D: 업무문서

목적: 담당자, 기한, 산출물 누락을 보여준다.

```text
관련 내용을 빠르게 검토한 뒤 적절한 시점에 공유하겠습니다.
시장 반응은 전반적으로 긍정적인 편이며, 추후 개선 방향을 논의할 수 있습니다.
필요한 부분은 내부 검토 후 정리하겠습니다.
```

기대 리포트:

- 제출 안정성: 우선 확인
- PDF 리포트에서 담당자/기한/산출물 보완 제안 표시

## 8. 데모 진행 순서

```text
랜딩
→ 프로토타입 고지 설명
→ 샘플 문서 선택
→ 검사 실행
→ 리포트 4축 설명
→ 문장별 근거 확인
→ 개선 제안 비교
→ 체크리스트
→ PDF 리포트
→ 문서함/설정 정책 설명
```

## 9. 피드백 질문

| 질문 | 목적 |
|---|---|
| 4축 리포트가 단일 점수보다 이해하기 쉬웠나요? | 리포트 구조 검증 |
| 문장별 근거가 충분히 구체적이었나요? | 근거 UX 검증 |
| 개선 제안이 실제 수정에 도움이 될 것 같나요? | 제안문 가치 검증 |
| PDF 리포트를 저장하거나 공유할 이유가 있나요? | PDF 가치 검증 |
| 데이터 보관 안내가 충분히 명확했나요? | 정책 이해도 검증 |

## 10. 금지 데모 방식

- 실제 사용자 원문 붙여넣기 유도
- 분석 정확도 검증으로 설명
- 작성자 판정 도구처럼 소개
- 민감한 자소서, 과제, 업무문서 입력
- 결과를 제출 가능 여부의 근거로 안내
