## CHECK-001 문서 검사 입력

| 항목 | 내용 |
|---|---|
| 목적 | 문서 유형과 본문 입력 후 검사 실행 |
| 진입 경로 | `/check` |
| 권한 | 비회원 |
| 주요 CTA | `리스크 점검하기` |

### 영역 구성

1. 페이지 타이틀
2. 문서 유형 선택
3. 텍스트 입력 영역
4. 글자 수/권장 기준
5. 보관정책 안내
6. 검사 실행 버튼

### 필드

| 필드 | 유형 | 기본값 | 설명 |
|---|---|---|---|
| 문서 유형 | Card/Select | 자소서 또는 기타 | 자소서, 리포트, 블로그, 업무문서, 기타 |
| 본문 | Textarea | 빈 값 | 검사할 문서 입력 |
| 검사 옵션 | Checkbox | 기본 전체 선택 | MVP에서는 표시만 가능 |
| 원문 보관 | Notice | 미보관 안내 | 실제 저장하지 않는 mock |

### 에러 메시지

| 조건 | 메시지 |
|---|---|
| 빈 입력 | 검사할 문서를 입력해 주세요. |
| 200자 미만 | 짧은 문서는 분석 신뢰도가 낮을 수 있습니다. |
| 10,000자 초과 | 한 번에 검사할 수 있는 글자 수를 초과했습니다. |

## CHECK-002 검사 진행 상태

| 항목 | 내용 |
|---|---|
| 목적 | 분석 중임을 사용자에게 안내 |
| 진입 경로 | 검사 실행 후 |
| 권한 | 비회원 |

### 표시 항목

- 문장 구조 확인 중
- 문체 자연도 분석 중
- 출처·인용 리스크 확인 중
- 제출 안정성 리포트 생성 중

### 구현 방식

- 별도 route `/checking` 또는 검사 화면 내 overlay 가능
- 1~2초 mock loading 후 `/report` 이동 가능

## REPORT-001 종합 리포트

| 항목 | 내용 |
|---|---|
| 목적 | 전체 리스크와 핵심 확인 항목 요약 |
| 진입 경로 | `/report` |
| 권한 | 비회원 |

### 영역 구성

1. 리포트 헤더
2. 절대 판정 아님 안내
3. 종합 리스크 카드
4. 4축 리스크 카드
5. 우선 확인 문장 목록
6. 저장/다운로드/재검사 액션

### 주요 문구

```text
이 결과는 작성자를 단정하지 않습니다.
문체와 문서 패턴을 바탕으로 제출 전 검토가 필요한 신호를 보여주는 참고 지표입니다.
```

### 4축 카드

| 카드 | 설명 |
|---|---|
| 문체 자연도 | 반복 구조, 과도한 정돈, 문장 길이 |
| AI 작성 리스크 | AI 생성 또는 AI 편집 가능성 신호 |
| 출처·인용 리스크 | 주장 대비 출처 부족, 인용 누락 |
| 제출 안정성 | 글 길이, 일관성, 증빙 가능성 |

## REPORT-002 문장별 분석

| 항목 | 내용 |
|---|---|
| 목적 | 위험 신호가 발견된 문장의 근거 확인 |
| 진입 경로 | 리포트 내 탭/섹션 |
| 권한 | 비회원 |

### 영역 구성

- 왼쪽/상단: 문서 본문 하이라이트
- 오른쪽/하단: 선택 문장 상세 패널

### 하이라이트 라벨

| 라벨 | 의미 |
|---|---|
| 문체 | 문체 자연도 점검 필요 |
| AI 리스크 | AI 작성/편집 가능성 신호 |
| 출처 | 출처 또는 인용 보완 필요 |
| 안정성 | 제출 전 구조 보완 필요 |

### 상세 패널

- 선택 문장
- 위험 유형
- 근거 설명
- 신뢰도
- 권장 조치
- 개선 제안 이동 버튼

## REPORT-003 개선 제안

| 항목 | 내용 |
|---|---|
| 목적 | 위험 문장에 대한 개선 방향 제공 |
| 진입 경로 | 리포트 내 탭/섹션 |
| 권한 | 비회원 |

### 구성

| 영역 | 내용 |
|---|---|
| 원문 | 사용자가 입력한 문장 |
| 개선 방향 | 왜 개선이 필요한지 |
| 제안문 | 더 자연스럽거나 출처 보완을 유도하는 문장 |
| 액션 | 복사, 적용 mock |

### 주의

- 탐지 점수를 낮춘다는 표현 금지
- 의미 보존과 명확성 개선 중심
- 출처가 필요한 문장은 “출처를 추가해 주세요”로 안내

## ERROR-001 공통 오류 상태

| 오류 | 처리 |
|---|---|
