# 데이터 정책 카피 스펙

## 1. 목적

이 문서는 사용자 화면에 표시할 데이터 처리, 보관, 삭제, PDF, 외부 데모 관련 문구를 확정하기 위한 copy spec이다.

정책 문구는 사용자가 과도하게 불안해하지 않으면서도 원문 처리 범위를 정확히 이해할 수 있어야 한다.

## 2. 공통 카피 원칙

| 원칙 | 기준 |
|---|---|
| 단정 금지 | 작성자, 작성 도구, 제출 가능성을 단정하지 않음 |
| 보관 범위 명확화 | 원문 저장 여부와 저장 위치를 화면마다 표시 |
| 짧은 문장 | 한 문장 60자 안팎 권장 |
| 동의 분리 | 원문 저장, PDF 생성, 조직 열람은 각각 별도 고지 |
| 복구 가능성 명확화 | 삭제 후 복구 가능 여부를 삭제 전 표시 |

## 3. 공통 고지 문구

| 위치 | 문구 |
|---|---|
| 리포트 상단 | `이 결과는 제출 전 확인을 돕는 참고 지표입니다. 작성자나 작성 도구를 단정하지 않습니다.` |
| 분석 신뢰도 | `문서가 짧아 일부 신호는 제한적으로 해석해야 합니다.` |
| 원문 기본 처리 | `기본 설정에서는 입력 원문을 저장하지 않습니다.` |
| 샘플 데모 | `이 화면은 샘플 문서로 제품 흐름을 확인하는 데모입니다.` |

## 4. `/check` 입력 화면

| 상태 | 문구 |
|---|---|
| 기본 안내 | `검사할 문서를 붙여넣고 문서 유형을 선택하세요.` |
| 보관 안내 | `입력한 문서는 현재 점검을 위해서만 사용되며, 기본 설정에서는 저장되지 않습니다.` |
| 짧은 입력 안내 | `200자 이상 입력하면 문장별 근거를 더 안정적으로 확인할 수 있습니다.` |
| 빈 입력 오류 | `검사할 문서를 먼저 입력해 주세요.` |
| 20자 미만 오류 | `문서가 너무 짧아 점검할 수 없습니다. 문장을 조금 더 입력해 주세요.` |
| 10,000자 초과 오류 | `한 번에 점검할 수 있는 길이를 넘었습니다. 문서를 나누어 입력해 주세요.` |

## 5. `/checking` 처리 중 화면

| 상태 | 문구 |
|---|---|
| 기본 | `문서의 문장 구조와 제출 전 확인 항목을 정리하고 있습니다.` |
| 보조 | `결과는 참고 지표로 제공되며, 최종 수정 여부는 직접 확인해 주세요.` |
| 오류 | `점검을 완료하지 못했습니다. 입력 내용을 확인한 뒤 다시 시도해 주세요.` |

처리 중 화면에서는 원문이 서버에 저장된다는 인상을 주는 표현을 사용하지 않는다.

## 6. `/report` 리포트 화면

| 위치 | 문구 |
|---|---|
| 종합 고지 | `이 리포트는 제출 전 확인할 신호를 정리한 참고 자료입니다.` |
| AI 작성 리스크 설명 | `이 항목은 작성 패턴 점검 신호이며, 작성자나 작성 도구를 판단하지 않습니다.` |
| 출처·인용 설명 | `수치, 조사, 일반화 주장에 근거 표시가 필요한지 확인합니다.` |
| 제출 안정성 설명 | `목적, 대상, 기한, 역할, 결과처럼 제출 전 빠지기 쉬운 항목을 확인합니다.` |
| 짧은 문서 경고 | `문서 길이가 짧아 결과를 보수적으로 확인해 주세요.` |

| 자소서 | `실제 경험, 본인 역할, 결과를 직접 넣어 완성해 주세요.` |

| 민감정보 | `개인정보가 포함될 수 있습니다. 제출 전 삭제 또는 마스킹을 확인해 주세요.` |

## 8. PDF 저장 화면

브라우저 PDF 저장을 기본 동작으로 둔다. 서버 PDF는 QA/제한 기능이므로 일반 사용자 화면에서는 보조 동작으로 둔다.

| 위치 | 문구 |
|---|---|
| 기본 버튼 | `PDF로 저장` |
| 보조 버튼 | `PDF 생성 확인` |
| 저장 전 안내 | `PDF에는 리포트 요약, 제한된 문장 근거, 개선 제안, 체크리스트가 포함됩니다.` |
| 원문 제한 안내 | `원문 전체는 기본 포함되지 않습니다.` |
| 민감정보 경고 | `PDF를 공유하기 전 연락처, 주소, 소속 등 민감정보가 남아 있는지 확인해 주세요.` |
| 데모 PDF 고지 | `이 PDF는 샘플 문서 기준의 프로토타입 데모 리포트입니다.` |

민감정보 신호가 감지되면 경고만 표시하는 것이 아니라 마스킹 권장 액션을 함께 제공한다. 고유식별번호처럼 위험도가 높은 패턴은 PDF 포함 전 우선 확인 상태로 막는 방향을 기본값으로 둔다.

| 전체 삭제 | `저장 데이터를 모두 삭제할까요?` | `현재 브라우저에 저장된 리포트와 설정을 모두 지웁니다.` | `모두 삭제` |
| 원문만 삭제 | `원문만 삭제할까요?` | `리포트 요약은 유지되지만 원문 비교 일부가 제한될 수 있습니다.` | `원문 삭제` |
| 설정 초기화 | `설정을 초기화할까요?` | `보관 방식과 화면 설정이 기본값으로 돌아갑니다. 저장 리포트는 별도로 삭제해야 합니다.` | `초기화` |

## 14. 외부 데모 피드백 폼 고지

```text
이 피드백 폼에는 실제 문서 원문, 이름, 연락처, 제출 예정 문서를 입력하지 마세요.
샘플 문서 데모를 본 뒤 화면 이해도와 개선 의견만 남겨 주세요.
```

## 15. 금지 카피

- 작성자를 단정하는 표현
- 제출 결과를 보장하는 표현
- 회피 목적처럼 읽히는 표현
- 감시, 적발, 판정처럼 불이익 판단을 떠올리게 하는 B2B 표현
- 실제 원문을 서버에 안전하게 저장한다고 단정하는 표현

## 16. 승인 조건

이 문서가 승인되면 다음 작업에 반영한다.

- `/check`, `/checking`, `/report`, `/pdf-report`, `/documents`, `/settings`, `/org` 화면 카피 정리
- PDF 저장 버튼 hierarchy 확정
- 삭제/원문 보관 모달 문구 정리
- 외부 데모 피드백 폼 고지 문구 적용
