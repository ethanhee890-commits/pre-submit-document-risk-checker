# 수용 기준 v0.1

## 1. MVP 완료 기준

덱스 작업은 아래 조건을 모두 만족해야 MVP 1차 완료로 본다.

| 영역 | 수용 기준 |
|---|---|
| 실행 | 로컬에서 앱 또는 HTML을 실행할 수 있다. |
| 확인 | 에던이 브라우저에서 주요 화면을 확인할 수 있다. |
| 흐름 | 랜딩 → 입력 → 검사 → 리포트 → 문장별 분석 → 체크리스트 흐름이 가능하다. |
| Mock | 실제 AI 모델 없이 mock 데이터로 리포트가 표시된다. |
| 정책 | 금지 문구가 UI에 없다. |
| 상태 | empty/loading/error/success 상태가 최소 구현되어 있다. |
| 반응형 | 모바일 주요 화면이 깨지지 않는다. |
| 보고 | 실행 방법, URL/경로, QA 결과, 변경 파일 요약이 제공된다. |

## 2. 화면별 수용 기준

### LAND-001

- 제품명이 표시된다.
- 제품이 `제출 전 문서 리스크 점검` 서비스로 설명된다.
- CTA 클릭 시 검사 화면으로 이동한다.
- 4대 점검 항목이 표시된다.
- 회피성 표현이 없다.

### CHECK-001

- 문서 유형을 선택할 수 있다.
- 텍스트를 입력할 수 있다.
- 글자 수가 표시된다.
- 빈 입력 시 오류가 표시된다.
- 검사 실행 시 로딩 후 리포트로 이동한다.

### REPORT-001

- 결과가 절대 판정이 아니라는 안내가 표시된다.
- 종합 리스크가 표시된다.
- 문체, AI 작성 리스크, 출처·인용, 제출 안정성 카드가 표시된다.
- 저장/다운로드/재검사 액션이 존재한다.

### REPORT-002

- 문장별 위험 구간이 표시된다.
- 문장을 클릭하면 상세 근거가 갱신된다.
- 위험 라벨이 표시된다.
- 권장 조치가 표시된다.

### REPORT-003

- 원문과 개선 제안이 구분 표시된다.
- 복사 버튼 클릭 시 토스트가 표시된다.
- 개선 제안 문구가 회피 목적이 아니다.

### REPORT-004

- 체크리스트 항목을 체크할 수 있다.
- 완료율 또는 완료 상태가 반영된다.

## 3. 완료 보고 수용 기준

덱스 완료 보고에는 아래가 반드시 포함되어야 한다.

```text
산출물 위치
실행 방법
확인 URL 또는 HTML 경로
route list
구현 화면 목록
미구현 화면 목록
주요 변경 파일
QA 결과
Git diff 또는 변경 요약
가정한 내용
알려진 이슈
```

## 4. 불합격 기준

아래 항목 중 하나라도 있으면 1차 완료로 보지 않는다.

- 실행 방법이 없거나 실행되지 않음
- 브라우저 확인 경로가 없음
- 빈 입력 오류가 동작하지 않음
- 리포트 상단에 참고 지표 안내가 없음
- `통과`, `우회`, `bypass`, `humanizer`, `100% 사람 글` 등 금지 문구 사용
- 문장별 하이라이트 또는 상세 근거가 없음
- 모바일에서 주요 CTA 또는 입력창이 깨짐
- QA 결과가 없음
