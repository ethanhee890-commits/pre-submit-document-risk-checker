# 보관/삭제 플로우 설계

## 1. 목적

이 문서는 문서함, 리포트 저장, 원문 보관, 삭제 플로우를 실제 사용자 파일럿 전에 확정하기 위한 설계안이다.

승인 전에는 사용자 원문 서버 저장 기능을 구현하지 않는다.

## 2. 보관 대상

| 대상 | 개인 MVP 기본값 | B2B 후보 |
|---|---|---|
| 리포트 요약 | 저장 가능 | 저장 가능 |
| 원문 본문 | 기본 미보관 | 조직 정책에 따라 선택 |
| PDF 파일 | 다운로드만 제공 | 다운로드/공유 링크 검토 |
| 체크리스트 상태 | 현재 화면 또는 저장 리포트 | 사용자별 저장 |
| 감사 로그 | 없음 | 관리자용 제공 |

## 3. 개인 사용자 플로우

### 3.1 리포트만 저장

```text
리포트 화면
→ 리포트 저장
→ "원문은 저장하지 않고 리포트만 저장합니다" 안내
→ 문서함에 리포트 요약 저장
```

저장 항목:

- 리포트 ID
- 문서 유형
- 4축 리스크 결과
- 문장별 근거
- 개선 제안
- 생성 시각
- 원문 미포함 플래그

### 3.2 원문 함께 저장

```text
설정에서 원문 함께 보관 켜기
→ 민감정보 안내
→ 리포트 저장
→ 원문 포함 저장
```

필수 조건:

- 기본값은 꺼짐
- 저장 전 재확인
- 문서함 카드에 원문 포함 여부 표시
- 삭제 시 원문도 함께 삭제

### 3.3 삭제

```text
문서함
→ 리포트 카드
→ 삭제
→ 확인 모달
→ 리포트 및 연결 원문 삭제
→ 삭제 완료 토스트
```

## 4. 자동 삭제 기준

| 항목 | 기본 권장 |
|---|---|
| 리포트 보관 | 7일 |
| 원문 보관 | 기본 미보관, opt-in 시 7일 |
| 만료 예정 표시 | 만료 1일 전 |
| 만료 리포트 정리 | 사용자가 직접 실행 또는 자동 정리 |

## 5. 삭제 UX 문구

```text
이 리포트를 삭제하면 저장된 리포트와 연결된 원문이 함께 삭제됩니다.
삭제 후에는 복구할 수 없습니다.
```

원문이 없는 경우:

```text
이 리포트에는 원문이 포함되어 있지 않습니다.
삭제하면 리포트 요약만 삭제됩니다.
```

## 6. 서버 저장 도입 시 API 초안

아래는 설계용 초안이며, 승인 전 구현하지 않는다.

```text
POST /api/reports
GET /api/reports
GET /api/reports/:id
DELETE /api/reports/:id
DELETE /api/reports
PATCH /api/settings/retention
```

## 7. 삭제 검증 기준

| 검증 | 기준 |
|---|---|
| 리포트 삭제 | 문서함 목록에서 제거 |
| 원문 삭제 | 원문 포함 저장 시 함께 제거 |
| 전체 삭제 | 사용자 저장 데이터 전부 제거 |
| 만료 삭제 | 만료 상태 리포트만 제거 |
| 삭제 안내 | 삭제 전 영향 범위 표시 |

## 8. B2B 확장 고려

| 항목 | 요구 |
|---|---|
| 조직 보관기간 | 관리자 설정 |
| 사용자 삭제 권한 | 역할별 제한 |
| 관리자 삭제 | 감사 로그 기록 |
| 법적 보존 | 별도 정책 필요 |
| 내보내기 | 조직 정책 승인 후 제공 |

## 9. 승인 전 금지 사항

- 원문 서버 저장 구현
- 계정 기반 문서함 구현
- 조직 관리자 원문 열람 구현
- 삭제 불가능한 저장 기능 구현
