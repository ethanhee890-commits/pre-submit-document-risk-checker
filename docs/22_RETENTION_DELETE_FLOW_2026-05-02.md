# 보관/삭제 플로우 설계

## 1. 목적

이 문서는 문서함, 리포트 저장, 원문 보관, 삭제 플로우를 실제 사용자 파일럿 전에 확정하기 위한 설계안이다.

승인 전에는 사용자 원문 서버 저장 기능을 구현하지 않는다.

## 2. 보관 대상

| 대상 | 개인 MVP 기본값 | B2B 후보 |
|---|---|---|
| 리포트 요약 | 저장 가능 | 저장 가능 |
| 원문 본문 | 기본 미보관 | 조직 정책에 따라 선택 |
| PDF 파일 | 다운로드만 제공 | 다운로드/공유 링크 검토 |
| 체크리스트 상태 | 현재 화면 또는 저장 리포트 | 사용자별 저장 |
| 감사 로그 | 없음 | 관리자용 제공 |

## 3. 보관 위치별 플로우

### 3.1 브라우저 임시 문서함

```text
리포트 화면
→ 리포트 저장
→ localStorage에 리포트 요약 저장
→ 문서함 카드 표시
```

특징:

- 서버 계정 없이 현재 브라우저에만 보관한다.
- 기본 저장 항목은 리포트 요약, 4축 결과, 체크리스트 상태, 생성 시각이다.
- 원문은 기본 저장하지 않는다.
- 브라우저 데이터 삭제 시 함께 삭제될 수 있음을 안내한다.

### 3.2 향후 서버 문서함

```text
로그인
→ 저장 동의
→ 서버 리포트 저장
→ 문서함 목록/상세에서 조회
```

특징:

- 데이터 정책 v1, 보관 기간, 삭제 권한 승인 후 별도 구현한다.
- 원문 포함 저장은 별도 opt-in과 민감정보 안내가 필요하다.
- 계정 삭제, 조직 권한, 감사 로그 정책과 함께 설계한다.

## 4. 개인 사용자 플로우

### 4.1 리포트만 저장

```text
리포트 화면
→ 리포트 저장
→ "원문은 저장하지 않고 리포트만 저장합니다" 안내
→ 문서함에 리포트 요약 저장
```

저장 항목:

- 리포트 ID
- 문서 유형
- 4축 리스크 결과
- 문장별 근거
- 개선 제안
- 생성 시각
- 원문 미포함 플래그

### 4.2 원문 함께 저장

```text
설정에서 원문 함께 보관 켜기
→ 민감정보 안내
→ 리포트 저장
→ 원문 포함 저장
```

필수 조건:

- 기본값은 꺼짐
- 저장 전 재확인
- 문서함 카드에 원문 포함 여부 표시
- 카드에는 원문 포함 표시만 노출하고 원문 내용은 기본 접힘 상태 유지
- 원문 상세 열람은 명시적 버튼으로만 제공
- 삭제 시 원문도 함께 삭제

### 4.3 삭제

```text
문서함
→ 리포트 카드
→ 삭제
→ 확인 모달
→ 리포트 및 연결 원문 삭제
→ 삭제 완료 토스트
```

## 5. 삭제 케이스

| 케이스 | 범위 | 확인 문구 |
|---|---|---|
| 단일 리포트 삭제 | 선택 리포트와 연결 원문 | 삭제 후 복구 불가 안내 |
| 전체 삭제 | 문서함의 모든 저장 리포트 | 저장 데이터 전체 삭제 안내 |
| 설정 초기화 | 보관 설정, 체크 상태, 화면 선호 | 저장 리포트 삭제 여부 분리 안내 |
| 원문 포함 데이터만 삭제 | 리포트는 유지, 원문만 제거 | 리포트 요약은 남고 원문은 제거됨 안내 |
| 만료 대상 정리 | 사용자가 선택한 정리 대상 | 정리 개수와 대상 범위 안내 |

## 6. 삭제 완료 증거

| 증거 | 기준 |
|---|---|
| 토스트 | "삭제가 완료되었습니다" 또는 원문만 삭제 완료 메시지 |
| 목록 개수 감소 | 문서함 요약 카운트 즉시 갱신 |
| 상세 URL 접근 | 삭제된 리포트 상세 진입 시 없음 상태 표시 |
| 원문 포함 표시 제거 | 원문만 삭제한 경우 카드 표시가 "원문 미포함"으로 변경 |
| localStorage 확인 | QA에서 저장 키가 제거되거나 개수가 감소 |

## 7. 자동 삭제 기준

MVP 기본값은 자동 삭제가 아니라 사용자가 직접 정리하는 방식으로 둔다. 실제 만료 자동 삭제는 사용자 신뢰와 복구 요구를 확인한 뒤 적용한다.

| 항목 | Sprint 3 권장 |
|---|---|
| 리포트 보관 | 수동 정리 우선 |
| 원문 보관 | 기본 미보관, opt-in 시 수동 삭제 제공 |
| 만료 예정 표시 | 향후 보관 기간 설정 승인 후 제공 |
| 만료 리포트 정리 | 사용자가 버튼을 눌러 직접 실행 |

## 8. 삭제 UX 문구

```text
이 리포트를 삭제하면 저장된 리포트와 연결된 원문이 함께 삭제됩니다.
삭제 후에는 복구할 수 없습니다.
```

원문이 없는 경우:

```text
이 리포트에는 원문이 포함되어 있지 않습니다.
삭제하면 리포트 요약만 삭제됩니다.
```

원문만 삭제하는 경우:

```text
리포트 요약은 유지하고 저장된 원문만 삭제합니다.
이후에는 문장 비교 일부가 제한될 수 있습니다.
```

## 9. 서버 저장 도입 시 API 초안

아래는 설계용 초안이며, 승인 전 구현하지 않는다.

```text
POST /api/reports
GET /api/reports
GET /api/reports/:id
DELETE /api/reports/:id
DELETE /api/reports
DELETE /api/reports/:id/original
PATCH /api/settings/retention
```

## 10. 삭제 검증 기준

| 검증 | 기준 |
|---|---|
| 리포트 삭제 | 문서함 목록에서 제거 |
| 원문 삭제 | 원문 포함 저장 시 함께 제거 |
| 전체 삭제 | 사용자 저장 데이터 전부 제거 |
| 원문만 삭제 | 원문 포함 표시 제거, 리포트 요약 유지 |
| 설정 초기화 | 설정값 기본값 복귀 |
| 삭제 안내 | 삭제 전 영향 범위 표시 |

## 11. B2B 확장 고려

| 항목 | 요구 |
|---|---|
| 조직 보관기간 | 관리자 설정 |
| 사용자 삭제 권한 | 역할별 제한 |
| 관리자 삭제 | 감사 로그 기록 |
| 법적 보존 | 별도 정책 필요 |
| 내보내기 | 조직 정책 승인 후 제공 |

권한 충돌 예시:

| 상황 | 기본 처리 |
|---|---|
| 제출자가 삭제 요청했지만 조직 보존기간이 남아 있음 | 사용자 화면에서는 삭제 요청 접수, 실제 삭제 가능일 안내 |
| 검토자가 리포트 삭제를 시도 | 권한 없음 안내, 관리자에게 요청 |
| 관리자가 삭제했지만 제출자가 PDF를 이미 다운로드 | 서비스 내 저장 데이터 삭제로 범위 제한 안내 |

## 12. 승인 전 금지 사항

- 원문 서버 저장 구현
- 계정 기반 문서함 구현
- 조직 관리자 원문 열람 구현
- 삭제 불가능한 저장 기능 구현
